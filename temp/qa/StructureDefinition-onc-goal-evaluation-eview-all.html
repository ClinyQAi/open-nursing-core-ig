Type of observation (code / type)
Cardinality: 1..1
CodeableConcept
 Describes what was observed. Sometimes this is called the observation "name".

 value must match SNOMED CT: 390906007 "Follow-up assessment" from LOINC Codes (>1000 codes)
Actual result
Cardinality: 0..1
CodeableConcept
 The information determined as a result of making the observation, if the information has a simple value.

 value is bound to Goal Evaluation Value Set. Codes:
  • SNOMED CT: 385652002 "Started (qualifier value)"
  • SNOMED CT: 385651009 "In progress (qualifier value)"
  • SNOMED CT: 255609007 "Partial"
  • SNOMED CT: 723510000 "Entered in error (qualifier value)"
Comments about the observation
Cardinality: 0..1
Annotation
 Comments about the observation or the results.
Protocol
Related measurements the observation is made from
Cardinality: 0..*
Reference
 The target resource that represents a measurement from which this observation value is derived. For example, a calculated anion gap or a fetal measurement based on an ultrasound image.

 value must point to a Observation
Context
Who and/or what the observation is about
Cardinality: 0..1
Reference
 The patient, or group of patients, location, or device this observation is about and into whose record the observation is placed. If the actual focus of the observation is different from the subject (or a sample of, part, or region of the subject), the focus element or the code itself specifies the actual focus of the observation.

 value must point to a Patient
What the observation is about, when it is not about the subject of record
Cardinality: 1..1
Reference
 The actual focus of an observation when it is not the patient of record representing something or someone associated with the patient such as a spouse, parent, fetus, or donor. For example, fetus observations in a mother's record. The focus of an observation could also be an existing condition, an intervention, the subject's diet, another observation of the subject, or a body structure such as tumor or implanted device. An example use case would be using the Observation resource to capture whether the mother is trained to change her child's tracheostomy tube. In this example, the child is the patient of record and the mother is the focus.

 value must point to a ONC Nursing Goal

Observation Goal Reference

Extension
 Extension to link goal evaluation observations to the patient goals being evaluated.

 value must be a Observation Goal Reference
